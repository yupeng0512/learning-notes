## 🎯 解决的核心痛点

| 痛点 | 症状 | Superpowers 解决方案 |
|------|------|---------------------|
| **每次重新调教** | 新 Session 从头讲背景需求 | 技能自动触发，流程内置 |
| **重复复述** | 同一调试流程反复解释 | 标准化调试流程 |
| **目标偏离** | Bug 修着修着偏了 | 计划驱动，任务检查点 |
| **遗漏检查** | 提交前漏测试全靠人脑记 | Pre-commit 强制验证 |
| **无纪律性** | AI 被动执行，缺乏主动引导 | AI 主动用斜杠命令引导工作 |

**核心问题诊断**：

> "问题不在 Claude Code，而在于没有稳定工作流。"

## 🔑 核心设计

### 技能分类

```
Superpowers 技能体系
├── 🧠 规划 & 头脑风暴（Planning & Brainstorming）
│   └── 在编码前完善想法、探索方案
│
├── ⚙️ 开发与重构（Development）
│   ├── 计划驱动开发
│   └── 子代理任务分派
│
├── 🧪 测试（Testing）
│   └── TDD 红绿重构流程
│
├── 🐞 调试（Debugging）
│   └── 系统化调试流程
│
├── 🤝 协作（Collaboration）
│   └── 代码审查流程
│
└── 🚀 提交前验证（Pre-commit checks）
    └── 强制验证测试
```

### 完整工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                  Superpowers 工作流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 头脑风暴 ──→ 提问完善想法 → 探索方案 → 保存设计文档      │
│        ↓                                                    │
│  2. Git Worktrees ──→ 新分支隔离工作区 → 验证测试基线        │
│        ↓                                                    │
│  3. 编写计划 ──→ 分解任务 → 精确文件路径 → 验证步骤          │
│        ↓                                                    │
│  4. 子代理开发 ──→ 每任务派遣子代理 → 两阶段审查             │
│        ↓            (规范符合性 + 代码质量)                  │
│  5. TDD 流程 ──→ 写失败测试 → 写最小代码 → 测试通过 → 提交   │
│        ↓                                                    │
│  6. 代码审查 ──→ 按计划审查 → 按严重程度报告                 │
│        ↓                                                    │
│  7. 完成分支 ──→ 验证测试 → 合并/PR/保留/丢弃                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 💡 核心洞见

### 1. "流程大于提示"的设计哲学

| 维度 | 传统提示词 | Superpowers 流程 |
|------|-----------|-----------------|
| **执行稳定性** | 每次可能不同 | 强制执行，稳定一致 |
| **知识载体** | 存在用户脑中 | 内化为技能 |
| **触发方式** | 用户手动描述 | 合适时机自动触发 |
| **覆盖范围** | 单点任务 | 完整生命周期 |

> **类比**：传统提示词是"每次临场发挥"，Superpowers 是"按剧本演出"。

### 2. 软件工程智慧的 AI 化

| 工程实践 | 传统执行方式 | Superpowers 内化方式 |
|----------|-------------|---------------------|
| **TDD** | 人类自觉遵守 | 强制红绿重构流程 |
| **Code Review** | 人工审查 | 两阶段自动审查 |
| **系统化调试** | 经验依赖 | 标准化调试流程 |
| **任务分解** | 人工规划 | 自动分解+检查点 |

> "把人类工程师几十年总结出来的软件工程智慧，内化成了 AI 的本能。"

### 3. 从"被动执行者"到"主动引导者"

| 角色转变 | 之前 | 之后 |
|----------|------|------|
| **Claude 定位** | 被动等待指令 | 主动引导工作 |
| **工作流** | 用户驱动 | 技能自动触发 |
| **质量保证** | 用户负责 | 流程内置 |

### 4. Skill 作为 AI 的"App"

> "在 Claude Code 的体系里，Skill 就像是给 AI 安装的 App。"

这个类比揭示了 AI 工具化的发展方向：
- **基础能力**：模型本身的推理能力
- **扩展能力**：通过 Skill/Plugin 安装专业能力
- **工程化**：把流程固化为可复用的"App"
