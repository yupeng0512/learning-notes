# Humanizer-zh 精读笔记：去除 AI 写作痕迹的 24 种模式

> **来源**: [op7418/Humanizer-zh](https://github.com/op7418/Humanizer-zh)
> **类型**: Claude Code Skill
> **精读日期**: 2026-01-24

## 核心命题

> **AI 写作的本质问题不是「不对」，而是「太平均」——它总是选择统计上最可能的表达，而人类写作的魅力恰恰在于那些「不太可能」的选择。**

## 问题本质：LLM 的统计陷阱

```
LLM 写作机制
    │
    ├─→ 输入：上下文
    ├─→ 计算：下一个 token 的概率分布
    ├─→ 输出：统计上「最安全」的选择
    └─→ 结果：平滑、泛泛、无锋芒
```

**关键洞见**：LLM 使用统计算法猜测下一个 token，结果倾向于适用最广泛情况的统计上最可能结果。

这解释了为什么 AI 文本：
- 总是用「此外」「至关重要」这些高频连接词
- 总是凑成三段式（训练数据中最常见的结构）
- 总是给模糊的「未来展望」（最安全的结尾方式）

## 核心模式详解

### 模式 1：过度强调意义

**AI 思维**：「我要让这段话显得很重要」→ 堆砌「关键」「至关重要」「里程碑」

| 改写前 | 改写后 |
|--------|--------|
| 加泰罗尼亚统计局于 1989 年正式成立，**标志着**西班牙区域统计演变史上的**关键时刻** | 加泰罗尼亚统计局成立于 1989 年，负责收集和发布区域统计数据 |

**规则**：删掉「标志着」「关键时刻」「里程碑」「深远影响」，直接说事实。

### 模式 7：AI 高频词汇

**AI 思维**：「我需要一个连接词」→ 选择训练数据中出现频率最高的那个

| 高频 AI 词汇 | 人类替代方案 |
|-------------|-------------|
| 此外 | 直接删除，或用「还」 |
| 至关重要 | 重要 |
| 深入探讨 | 分析、讨论 |
| 不断演变的格局 | 具体说什么在变 |
| 充满活力的 | 删除或用具体描述 |

**检测技巧**：如果一篇文章「此外」出现 3 次以上，大概率是 AI 写的。

### 模式 9：否定式排比

**AI 思维**：「我要强调重要性」→ 「不仅仅是 X，而是 Y」

| 改写前 | 改写后 |
|--------|--------|
| **这不仅仅是**一次更新，**而是**我们思考生产力方式的**革命** | 更新添加了批处理和键盘快捷键 |

**问题本质**：用句式的气势掩盖内容的贫乏。真正重要的事不需要宣布「这很重要」。

### 模式 10：三段式法则

**AI 思维**：「我要显得全面」→ 「A、B 和 C」

| 改写前 | 改写后 |
|--------|--------|
| 它提供了**无缝、直观和强大的**用户体验 | 它比旧版快，界面也更简洁 |

**规则**：两个具体比三个抽象更有说服力。

### 模式 22-24：结尾三连

**AI 标准结尾公式**：
```
「尽管存在挑战」+「未来依然光明」+「期待进一步发展」
```

| 改写前 | 改写后 |
|--------|--------|
| 公司的未来看起来光明。激动人心的时代即将到来 | 该公司计划明年再开设两个地点 |

**规则**：删掉所有「展望」「期待」「光明」，用具体计划或数据收尾。

## 核心心法

### 五条规则

| 规则 | 说明 | 检测方法 |
|------|------|----------|
| **删除填充短语** | 「为了实现这一目标」→「为了」 | 数连接词数量 |
| **打破公式结构** | 避免三段式、否定排比 | 检查并列结构 |
| **变化节奏** | 长短句交错 | 数连续句子长度 |
| **信任读者** | 不解释隐喻，不软化观点 | 删除「换句话说」 |
| **删除金句** | 听起来像名言的就重写 | 问「能发朋友圈吗」 |

### 快速检查清单

```
✓ 连续三个句子长度相同？ → 打断一个
✓ 段落以简洁单行结尾？ → 变换方式
✓ 揭示前有破折号？ → 删除
✓ 使用了「此外」「然而」？ → 考虑删除
✓ 三段式列举？ → 改为两项或四项
```

## 「无灵魂」vs「有灵魂」

> **避免 AI 模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。**

| 无灵魂的写作 | 有灵魂的写作 |
|-------------|-------------|
| 每个句子结构相同 | 长短交错，节奏变化 |
| 只报道事实，没有观点 | 对事实有反应、有态度 |
| 不承认不确定性 | 「我真的不知道该怎么看」 |
| 从不用第一人称 | 适当用「我」「让我困扰的是」 |
| 没有幽默、没有锋芒 | 允许一些混乱和跑题 |

### 示例对比

**无灵魂**：
> 实验产生了有趣的结果。智能体生成了 300 万行代码。一些开发者印象深刻，另一些则持怀疑态度。影响尚不明确。

**有灵魂**：
> 我真的不知道该怎么看待这件事。300 万行代码，在人类大概睡觉的时候生成的。开发社区有一半人疯了，另一半人在解释为什么这不算数。真相可能在无聊的中间某处——但我一直在想那些通宵工作的智能体。

## 可迁移洞见

### 对写作的启示

| 启示 | 说明 |
|------|------|
| **具体胜过抽象** | 「批处理功能」比「强大的用户体验」有力 |
| **承认比回避好** | 「我不确定」比假装全知更可信 |
| **两个比三个好** | 并列两项比强凑三项更自然 |
| **删除比添加好** | 每个「此外」都可以删 |

### 对 AI 使用的启示

| 场景 | 策略 |
|------|------|
| 写初稿 | 用 AI 生成，然后人工「去味」 |
| 写正式文档 | 用这个 Skill 检查一遍 |
| 训练自己 | 对照 24 种模式，提升写作敏感度 |

## 延伸思考

1. **AI 检测的攻防**：如果所有人都用 Humanizer，AI 检测器会失效吗？
2. **风格迁移**：除了「去 AI 味」，能否「加人味」——模仿某个作家的风格？
3. **中英差异**：中文特有的 AI 痕迹是什么？
4. **逆向工程**：如果想让 AI 写得更像人，应该怎么 prompt？
