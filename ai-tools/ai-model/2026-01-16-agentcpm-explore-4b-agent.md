### 核心知识点

#### 关键概念

| 概念 | 定义 | 重要性 |
|------|------|--------|
| **端侧模型** | 可在手机、边缘设备等本地运行的小参数模型 | 决定 AI 能否真正普及 |
| **长程探索** | 智能体进行 100+ 轮不重复的环境交互 | Agent 核心能力指标 |
| **SFT 过拟合** | 小模型监督微调后"死记硬背"，泛化能力下降 | 小模型训练的核心瓶颈 |
| **RL 奖励偏差** | 长链路最后一步失败导致整条正确推理被惩罚 | 强化学习的信号噪声问题 |

#### 核心洞见

1. **参数量 ≠ 能力上限**：4B 可以战胜 30B+，关键在训练方法论
2. **小模型需要"专属训练策略"**：大模型的方法直接用在小模型上会失效
3. **信息预处理是小模型的"外脑"**：在信息进入模型前完成精炼过滤
4. **类人智能行为可训练习得**：质疑、求真、变通、执着

## 📚 核心问题：算力民主化的困境

### 为什么需要端侧智能体？

想象一个场景：你想在手机上运行一个能帮你完成复杂任务的 AI 助手（比如自动浏览网页、查资料、做调研）。

- **GPT-4 级别**：需要数据中心级算力，你只能通过 API 调用
- **70B 模型**：需要多块高端显卡，普通人无法本地运行
- **4B 模型**：可以在手机、笔记本、边缘设备上运行 ✅

## 🔧 三大关键技术

### 技术一：模型融合（解决 SFT 过拟合）

**问题**：小模型在 SFT 阶段极易"死记硬背"，仅调整 Prompt 中无关的工具描述，模型性能就会大幅下滑。

**方案**：
```
新模型 = α × 训练后专用模型 + (1-α) × 训练前通用模型
```

**原理**：
- 通专模型一致的泛化参数得以保留
- 互补的专业能力得以强化
- 因过拟合产生的随机噪音参数在融合中相互抵消

**效果**：性能提升约 7%

---

### 技术二：信号去噪（修正 RL 奖励偏差）

**问题**：
```
智能体执行 100 步任务：
  第 1-99 步：全部正确 ✅
  第 100 步：最后一步出错 ❌

传统 RL：整条轨迹 = -1（惩罚）
真相：前 99 步都是对的！
```

**方案**：
- 严格的奖励信号去噪
- 筛选真正具备策略更新价值的轨迹
- 对长步骤但最终失败的样本，不进行全轨迹惩罚

**效果**：避免负面信号污染已学到的正确推理逻辑，保护小模型脆弱的训练过程

---

### 技术三：信息精炼（对抗推理长文干扰）

**问题**：网页返回的冗长噪音对小模型影响极大。使用不同能力的模型对上下文进行摘要，最终 GAIA 性能差异可达 10%。

**方案**：
```
原始网页 → [信息精炼模块] → 精炼后的关键信息 → 4B 模型

信息精炼模块的工作：
1. 过滤噪音（广告、导航等）
2. 提取关键段落
3. 压缩冗余信息
4. 保留核心数据
```

**效果**：让小模型能聚焦于关键信息，而不是被噪音淹没

### AgentDock：工具沙盒统一管理调度平台

| 功能 | 价值 |
|------|------|
| 原生支持 16 个 MCP 服务 | 即插即用 |
| 100+ QPS 高并发 | 生产级性能 |
| 输出标准化 | 不同工具统一格式 |
| 自动重试 + 服务自愈 | 健壮容错 |
| 备用工具自动切换 | 高可用 |

### AgentRL：极简高效的异步强化学习框架

| 特性 | 传统框架 | AgentRL |
|------|----------|---------| 
| 代码量 | 数万行 | 7 个文件，1000+ 行 |
| 接入方式 | 深度定制 | 标准 ChatCompletions 接口 |
| 训采关系 | 同步阻塞 | 全异步流水线 |
| 长文本支持 | 困难 | 原生支持 128K+ |

### AgentToLeaP：智能体能力一键式评测平台

- 支持 GAIA、HLE 等 8 个主流榜单的一键测评
- 模块化扩展，可接入自定义测试集
- 一行命令即可启动全流程评测

|--------|----------------------|

### AI 智能体的部署演进

```
阶段1: 云端大模型（现在主流）
  用户 ──API──▶ 云端 GPT-4/Claude
  问题: 延迟、成本、隐私

阶段2: 端侧小模型（AgentCPM 证明可行）
  用户 ──本地──▶ 手机/PC 上的 4B 模型
  优势: 快速、免费、私密

阶段3: 端云协同（未来趋势）
  简单任务 → 端侧处理
  复杂任务 → 云端协助
```
