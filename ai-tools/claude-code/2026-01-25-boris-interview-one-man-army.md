## Boris 的「一人军团」工作模式

### 核心理念

```
我不会傻等着 AI 干活。
我会同时开 5-10 个任务窗口。
它们在干活，我喝咖啡。
```

### 多线程管理者模型

```
        ┌─ Claude 1: 整理收据
        ├─ Claude 2: 写代码
你 ──→  ├─ Claude 3: 做研究
        ├─ Claude 4: 发邮件
        └─ Claude 5: 催更 Slack
             ↓
        系统通知需要输入时切换
```

**核心转变**：从「打字员」→「多线程任务管理器」

## 访谈实战演示

### 第一关：收据整理（反向启发）

| 步骤 | 行为 |
|------|------|
| 1 | Boris 开放 Receipts 文件夹权限给 Cowork |
| 2 | 指令："按收据日期重命名" |
| 3 | Cowork 遇到模糊日期，**主动反问**："跳过还是你来定？" |
| 4 | 确认后，瞬间完成所有重命名 |

**亮点**：反向启发（Reverse Elicitation）—— AI 知道何时该问人

### 第二关：云端操作（跨平台自动化）

```
本地 CSV → "我不要本地的，搞个 Google Sheet"
    ↓
Cowork 自动：
  1. 打开 Chrome（需授权）
  2. 登录 Google Sheets
  3. 新建表格
  4. 填入数据
  5. 发现格式问题，自动修正
```

**亮点**：AI 自主控制浏览器 + 自我纠错

### 第三关：Slack 自动催更

| 场景 | 实现 |
|------|------|
| 监控项目进度表 | Cowork 定期检查 |
| 发现数据未填 | 自动 Slack 私聊对应工程师 |
| 效果 | 项目经理的重复工作完全自动化 |

## Boris 六条黄金法则

### 1. 影分身之术：并行是王道

```
❌ 盯着一个 AI 等它思考
✅ 同时运行 5-10 个任务
   - 终端
   - Web 端
   - 手机 App
```

**效果**：产出呈指数级增长

### 2. 拒绝「降智」：只用最聪明的模型

| 误区 | 真相 |
|------|------|
| 小模型省钱 | Opus 4.5 + Thinking 一次做对，反而省 Token |
| 快就是好 | 足够聪明 = 极少返工 = 最终更快 |

### 3. 团队「第二大脑」：claude.md

```markdown
# claude.md 示例

## 约定
- Claude 犯了错？写进来
- 特殊代码规范？写进来
- 项目背景？写进来

→ 每次干活前 Claude 都会先读这个文件
```

**本质**：可累积的团队知识库

### 4. 大召唤术：PR 中 @Claude

```
代码审查发现问题
    ↓
评论里 @Claude
    ↓
AI 自动修复 + 推送到分支
```

**效果**：系统随时间自动进化（复利工程）

### 5. 谋定而后动：计划模式

```
1. 进入计划模式
2. 与 Claude 讨论方案
3. 打磨到完美
4. 切换执行模式
5. 代码一步到位
```

> **只要计划是对的，代码就是对的。**
> (Once the plan is good, the code is good)

### 6. 给它一双「眼睛」：自我验证

| 做法 | 效果 |
|------|------|
| 安装 Chrome 扩展 | Claude 能看到浏览器页面 |
| 让它运行代码 | Claude 能自测 |
| 结果 | 产出质量「好得吓人」 |

**类比**：画家不能蒙眼画画，AI 也需要看到结果
