## 技术架构

### Agent 架构选型与设计 — 2026-02-25

> 来源: ai-agent-learning/notes/agent-architecture-design.md

- **核心判断**: 在需要生产级 AI Agent 的场景下，应先用「实习生标准」四维框架（执行/思考/进化/工程保障）做任务分析，再按任务复杂度、依赖关系和迭代需求选择 ReAct / Plan-Execute / Reflexion 的组合，因为任务特征与架构能力存在可归纳的映射关系，选型错误是失败的主要来源。
- **关键问题**: 这个任务的核心约束是什么？（成本 / 质量 / latency / 可解释性 / 可扩展性）
- **心智模型**: 系统思维 + 逆向思维
- **验证状态**: 待验证

### AI Agent 架构选型——简单性优先 — 2026-02-25

> 来源: Anthropic 构建有效 AI Agent 的实践指南

- **核心判断**: 在构建 AI 驱动的任务系统时，应默认从单次 LLM 调用或 Workflow 开始，只在任务步骤不可预测且复杂度能带来可衡量的性能提升时升级为 Agent；同时将至少与提示工程同等的精力投入工具接口（ACI）设计，因为工具格式和文档对 LLM 使用准确率的影响远超直觉。
- **关键问题**: 这个任务的步骤是可预测的吗？如果不可预测，复杂度提升能否被量化证明？
- **心智模型**: 奥卡姆剃刀 + 激励结构分析 + 概率思维
- **验证状态**: 待验证

### Skill 注入的倒 U 型增益法则——精准挂载胜于全面覆盖 — 2026-02-27

> 来源: SkillsBench (BenchFlow + Amazon/ByteDance/Foxconn + Stanford/CMU/UC Berkeley/Columbia/Oxford)

- **核心判断**: 向 Agent 注入外部程序性知识时，增益遵循倒 U 型曲线：收益 = f(领域稀缺度 × 结构精简度) — 超过阈值则负增益。应将 Skill 限制在 2-3 模块/任务，采用 Compact 结构，优先为模型知识稀缺的垂直领域编写。
- **关键问题**: 模型在这个领域的先验知识有多强？如果已经很强（如通用编程），注入 Skill 可能弊大于利。
- **心智模型**: 倒 U 型曲线 + 奥卡姆剃刀 + 瓶颈理论 + 概率思维
- **验证状态**: 论文实证已验证（7,308 次轨迹，84 任务），待个人实践验证

### 上下文是有限资源，精准胜于丰富 — 2026-02-25

> 来源: Anthropic Context Engineering + Manus 上下文工程六大原则

- **核心判断**: 上下文不是"越多越好"而是"越精越好"；KV-cache 命中率是核心性能指标，Context rot 随 n^2 膨胀导致聚焦能力衰减。
- **关键问题**: 这个 token 对当前决策的信息增益是多少？如果为零，它就不应该在上下文中。
- **心智模型**: 瓶颈理论 + 概率思维 + 激励结构
- **验证状态**: 部分验证于 InfoHunter、TruthSocial Monitor；SkillsBench 论文提供 7,308 次轨迹的大规模实证支持（2026-02-27 补充）

### Agent 工具设计的投入回报远超提示工程 — 2026-02-25

> 来源: Anthropic "Writing Effective Tools for Agents"

- **核心判断**: 应将至少 50% 的优化时间投入工具接口设计（ACI）；工具格式改进（Diff→完整文件）可使错误率降低 87.5%；用 Claude 优化 Claude 自己的工具（元循环自举）是最高效的优化路径。
- **关键问题**: 如果一个实习生只看你的工具文档，能否零歧义地正确使用？
- **心智模型**: 逆向思维（防呆 > 防错）+ 激励结构
- **验证状态**: 已验证于 Anthropic SWE-bench

## 风险决策

### 杠杆交易中的风险决策 — 2026-02-25

> 来源: trading-learning/sessions/2025-12-31/session-notes.md

- **核心判断**: 在杠杆交易中，应先设定并执行止损，再考虑仓位与杠杆；因为止损缺失是致命风险，杠杆只是放大结果，而多数人误把杠杆当作风险根源。
- **关键问题**: 这笔交易我愿意承担的最大亏损是多少？（开仓前必须能回答，否则不开仓）
- **心智模型**: 期望值思维 + 概率思维 + 反直觉校正
- **验证状态**: 待验证

### 经济周期决策——框架比预测更重要 — 2026-02-25

> 来源: trading-learning/sessions/2026-01-19/session-notes.md（美林投资时钟）

- **核心判断**: 应用经济周期框架的核心价值不在于精确预测，而在于建立"GDP + CPI → 四阶段 → 配置"的结构化思维，将模糊的市场感觉转化为可推理的决策链条。
- **关键问题**: 当前经济周期中，政策传导的方向和速度是什么？
- **心智模型**: 系统思维（负反馈螺旋）+ 概率思维（70% 符合经典周期）
- **验证状态**: 理论学习阶段

## 元认知

### 身份决定行为——改变视角比改变动作更重要 — 2026-02-25

> 来源: Dan Koe "How To Reinvent Your Life In One Day"

- **核心判断**: 应优先重构身份认同而非强制改变行为，因为身份是固化的视角，行为是视角的自然产物；拖延不是缺乏自律，而是在追求"保护自己免受评判"的隐藏目标。
- **关键问题**: 我正在追求的隐藏目标是什么？它与我声称的目标一致吗？
- **心智模型**: 控制论反馈环 + 逆向思维 + 心智发展阶段模型
- **验证状态**: 部分验证于交易学习（止损纪律需先重构身份认同）

## 工程管理

### AI 时代工程师角色转变——从实现者到验收者 — 2026-02-25

> 来源: 黄东旭 Vibe Engineering 2026 + Product Tri-Ownership 框架

- **核心判断**: 工程师核心价值从"写代码"转变为"需求制定 + 测试验收 + 模块治理"；不可复制的头狼模式应拆解为可培养的 PTO 三角色 + N 个 AI 施工队。
- **关键问题**: 你目前 90% 的时间花在哪里？如果答案是"写代码"，你可能在做 AI 更擅长的事。
- **心智模型**: 系统思维 + 逆向思维
- **验证状态**: 部分验证于 TiDB Rust 重写实践
